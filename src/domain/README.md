# Domain Layer

このディレクトリ（`src/domain`）は、Clean Architectureにおける**ドメイン層（Enterprise Business Rules）**に相当します。
システムの中核となるビジネスロジックやドメイン知識をカプセル化し、他のあらゆる層（DB、UI、フレームワークなど）への依存を持たないように設計されています。

## 主な構成要素

- **Entities (エンティティ)**:
  - 一意な識別子（ID）を持ち、ライフサイクルを通じた状態遷移のルールをカプセル化するクラスです。
  - 例: `Issue`（指摘）、`Project`（プロジェクト）。
  - 指摘のステータス変更（`OPEN` -> `IN_PROGRESS` -> `DONE`）などのビジネスルールは、エンティティのメソッドとして実装されます。
- **Value Objects (値オブジェクト)**:
  - 識別子を持たず、属性によってのみ等価性が定義されるオブジェクトです。
  - 例: 座標情報（`WorldPosition`）。
- **Domain Services**:
  - 特定のエンティティに属さない、複数のエンティティにまたがるビジネスロジックを配置します。
- **Repositories (Interface)**:
  - ドメインオブジェクトの永続化を行うためのインターフェース定義です。実際のアクセス実装はインフラストラクチャ層で行いますが、依存性の逆転の原則（DIP）によりインターフェースはドメイン層が所有します。

## 設計制約
- **他層への依存禁止**: ドメイン層内のコードは `src/application` や `src/infrastructure`、`src/app` 等を絶対に import してはいけません。
- **純粋なTypeScript**: 特定のフレームワークやライブラリに強く依存しない、プラットフォーム非依存なコードを記述します。
