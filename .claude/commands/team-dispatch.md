# 作業プラン → Agent Teams 分配プロンプト生成

作業プランファイル: $ARGUMENTS

## このスキルの目的

渡された作業プランファイルを読み込み、タスクの依存関係と並行実行可能性を分析して、
**エージェント別の分割プランファイル**を生成し、PMエージェントにそのまま渡せるプロンプトを出力する。

---

## 実行手順

### Step 1: 作業プランの読み込みと構造化

`$ARGUMENTS` で指定されたファイルを読み込み、以下を抽出する:

1. **タスク一覧**: 各タスクのID・内容・変更対象ファイル
2. **依存関係**: タスク間の前後関係（AがBの前提となる等）
3. **変更レイヤー**: 各タスクがどのアーキテクチャ層に属するか
   - Domain層: `src/domain/`
   - Infrastructure層: `src/infrastructure/`, `prisma/`
   - Application層: `src/application/`
   - Presentation層（API + UI）: `src/app/`
   - ドキュメント: `docs/`, `README.md`

---

### Step 2: エージェントマッピング分析

利用可能なエージェント（`subagent_type`）:

| subagent_type | 担当スコープ | ファイルオーナーシップ |
|---|---|---|
| `domain-architect` | 集約・VO・Repositoryインターフェース・ドメインエラー | `src/domain/**` |
| `infrastructure-engineer` | Prisma具象Repository・MinIO実装・Docker・マイグレーション | `src/infrastructure/**`, `prisma/**` |
| `application-engineer` | Command/Queryハンドラ・DTO | `src/application/**` |
| `frontend-engineer` | API Route Handlers・pages・components・APS Viewer | `src/app/**` |
| `doc-writer` | 設計ドキュメント・Mermaidダイアグラム | `docs/**`, `README.md` |
| `reviewer` | コードベース全体の読み取り調査・仕様差分分析・レビューレポート作成 | **読み取り専用**（全ファイル参照可、変更不可） |

各タスクを上記エージェントにマッピングする。1タスクが複数レイヤーにまたがる場合は分割する。

**分析観点:**
- 同一エージェントに割り当てられるタスクはグループ化する
- タスク数ゼロのエージェントは省略する
- ファイル編集の競合が起きないようにオーナーシップを厳密に分離する
- 同一 subagent_type でもファイルが競合しなければ複数チームに分割してよい
- `reviewer` は読み取り専用のため、ファイルオーナーシップの競合は発生しない。複数 reviewer チームの並行実行が常に可能
- 調査結果に基づく修正が必要な場合は、後続Waveで実装系エージェントに委譲する

---

### Step 3: 実行順序の決定

依存グラフを構築し、以下のルールで実行ウェーブ（波）を決定する:

```
Wave 1: 前提依存がないタスク群（通常 Domain層）
Wave 2: Wave 1 完了後に実行可能なタスク群（通常 Infrastructure + Application、並行）
Wave 3: Wave 2 完了後に実行可能なタスク群（通常 Frontend）
Wave 4: Wave 3 完了後（通常 Doc、任意）
```

**最適化ルール:**
- 同一Wave内のエージェントはすべて並行実行する
- Wave数は最小化する（不要な逐次化を避ける）
- Domain層に変更がなければ Wave 1 を省略し、Wave 2 から開始
- Presentation層のみの変更なら Wave 1 = frontend-engineer のみ（単一Wave）

---

### Step 4: エージェント別分割プランの生成

各エージェント（チーム）ごとに独立した作業プランファイルを生成する。
分割プランには、そのエージェントが作業を完遂するために必要な情報をすべて含める。

**分割プランのファイルパス:**
`C:\Users\prove\.claude\plans\{元プラン名}-{agent-type}.md`
（同一 subagent_type が複数チームの場合: `{元プラン名}-{agent-type}-{N}.md`）

**分割プランの構成（必ずこの順序で記述）:**
```
# {エージェント名} 作業プラン

## 前提条件
（このエージェントが作業を開始する前に完了している必要がある内容。なければ「なし（並行実行可能）」）

## タスク一覧
（具体的な変更内容。現状のコード・修正方針・コード例を含める）

## 変更対象ファイル
| ファイル | 操作 | 内容 |
|---------|------|------|
| ... | 新規/変更 | ... |

## 参照すべきファイル（読み取りのみ）
- ...

## 制約
- 担当ファイルのみ変更すること（他ファイル変更禁止）
- TypeScript strict mode: any型禁止
- （レイヤー固有の制約）

## 完了条件
- ...
```

**単一エージェントで完結する場合:**
分割プランは1ファイルのみ生成する。

---

### Step 5: 検証チェックリストの生成

元の作業プランの「検証方法」「完了条件」セクション、および各分割プランの「完了条件」から、
PMエージェントが全チーム完了後に実施すべき**検証チェックリスト**を生成する。

チェックリストには以下を含める:
1. **型チェック**: `npx tsc --noEmit` でエラーなし
2. **コードレビュー観点**: 各分割プランの完了条件を満たしているか（変更ファイルを実際に読んで確認）
3. **動作確認観点**: 元プランの検証方法に記載された確認項目（ロール別の画面遷移・表示確認など）
4. **アーキテクチャ制約**: 依存方向違反がないか（Domain層の外部依存importなど）

---

### Step 6: PMエージェント向けプロンプトの出力

分割プランファイルと検証チェックリストを組み合わせて、PMエージェントにそのまま渡せるプロンプトをチャットに出力する。

**出力形式:**

```
## Agent Teams 分配

### サマリー

| Wave | チーム | subagent_type | プランファイル |
|------|--------|--------------|---------------|
| 1 | Team 1 | {type} | {path} |
| 1 | Team 2 | {type} | {path} |
| 2 | Team 3 | {type} | {path} |

### PMエージェントへのプロンプト

以下の作業プランに基づいてAgent Teamsを作成・実行してください。

【Wave 1】（並行実行）
- Team 1: subagent_type=`{type}`, プロンプト=「{path} を読んで実行せよ」
- Team 2: subagent_type=`{type}`, プロンプト=「{path} を読んで実行せよ」

Wave 1 完了後 →

【Wave 2】（並行実行）
- Team 3: subagent_type=`{type}`, プロンプト=「{path} を読んで実行せよ」

【全チーム完了後 — レビュー・検証】
以下をすべて確認してください。問題があれば修正してください。

1. 型チェック: `npx tsc --noEmit` を実行し、エラーがないことを確認
2. コードレビュー:
   - {各分割プランの変更対象ファイルを読み、完了条件を満たしているか確認}
   - {アーキテクチャ制約違反がないか確認（src/domain/ の外部import等）}
3. 動作確認:
   - {元プランの検証方法から抽出した確認項目を箇条書きで列挙}
```

---

## 注意事項

- `$ARGUMENTS` が空の場合は、`C:\Users\prove\.claude\plans\` フォルダ内の `.md` ファイルのうち**最も更新日時が新しいもの**を自動選択する。Bashで `ls -t C:/Users/prove/.claude/plans/*.md | head -1` を実行して特定すること。該当ファイルがない場合はエラーメッセージを出力して終了する
- 作業プランファイルが見つからない場合もエラーメッセージを出力して終了する
- 元の作業プランの意図や優先度を変更しない。分配と並行化の最適化のみ行う
- 分割プランにはそのエージェントが担当しないファイルへの変更指示を含めない
- ファイルオーナーシップの競合が検出された場合は、ユーザーに確認を取る
- 分割プランには参照すべきファイルのパスを具体的に記載する
- PMエージェントへのプロンプトは簡潔に。詳細は分割プランファイルに委譲する
